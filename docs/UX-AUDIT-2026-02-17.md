# UX Audit Report: Houseworks
**Date:** 2026-02-17
**Auditor:** OpenClaw Agent
**Scope:** Browser-based visual and functional inspection (Desktop, Tablet, Mobile)
**Target:** http://localhost:3002

## 🚨 Executive Summary
The audit was partially blocked by a **Critical Functional Defect** in the data fetching layer. While the application allows authentication, the **Dashboard fails to load existing workspaces and boards**, displaying "0 Boards" and "No workspaces found" despite valid data in the database. Consequently, the **Board View** and **Item Views** are inaccessible via the UI.

Visual inspection was completed for **Authentication** (Sign In/Up) and **Dashboard** (Empty State).

## 🔴 Critical Severity
### 1. Data Fetching Failure (Functional Blocker)
- **View:** Dashboard / Sidebar
- **Observation:** After signing in as `admin@houseworks.local`, the dashboard reports "0 Boards" and "0 Items", and the sidebar shows "No workspaces found".
- **Evidence:** 
    - Seed data confirmed via CLI (`Post-Production` workspace exists).
    - Direct navigation to `/workspace/[id]/board` returns "Board not found".
    - **Root Cause Analysis:** The tRPC procedures (`dashboardStats`, `getById`) return `null` or empty arrays. This strongly suggests a mismatch between the `ctx.session.user.id` and the `ownerId` in the database, or a failure in the NextAuth session resolution in the tRPC context.
- **Impact:** **Catastrophic.** Users cannot access any core functionality beyond login.

### 2. Mobile Layout Collisions
- **View:** Dashboard (Mobile 375px)
- **Observation:** The "Workspace Management" section header collides with the "WORKSPACE / BOARD / TEAM" tabs and the Next.js dev tools widget.
- **Evidence:** Screenshot `2ef010eb` shows text overlapping and button misalignment.
- **Recommendation:** Increase vertical spacing between the section header and the tabs. Stack tabs vertically on mobile or use a dropdown if horizontal space is insufficient.

## 🟠 Major Severity
### 1. Board View Inaccessible
- **View:** Board Page
- **Observation:** Direct navigation to valid board URLs triggers a generic "Board not found" empty state.
- **Impact:** Cannot audit Board layout, Kanban columns, or Item cards.

### 2. Sign-In Page Feedback
- **View:** Sign In
- **Observation:** Submitting the form with a missing password provided no immediate field-level validation until the backend rejected it (or it failed silently/refreshed).
- **Recommendation:** Add client-side validation for required fields before submission.

## 🟡 Minor Severity & Suggestions
- **Navigation:** The "Need an invite? Return to the workspace" link on the Sign-in page points to `/`, which redirects unauthenticated users back to `/sign-in`. This creates a confusing loop.
- **Visuals:**
    - **Sidebar (Desktop):** The "Post-Production Hub" heading has good hierarchy, but the "No workspaces found" text in the sidebar has low contrast and visual weight, making it look like a disabled state rather than an empty state.
    - **Empty States:** The "Items by Status" card is completely empty ("No items yet") and visually unbalanced compared to the numerical cards next to it.
- **Interactions:** The "Notifications" dropdown works well but overlaps dashboard cards without a backdrop blur or dimming, which can make it visually busy against complex backgrounds.

## ⚠️ Unverified Views
The following views could not be inspected due to the Critical Data Fetching Failure:
1. **Board Kanban View** (Columns, Cards, Drag-and-Drop)
2. **Item Detail Modal/Panel**
3. **Workspace Settings**
4. **User Profile**
