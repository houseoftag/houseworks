### D3. Universal @mentions [FUTURE]
**What:** In all comments, notes, descriptions — type @ to mention a workspace member, triggering a notification with link to the relevant item.
**Architecture plan:** Single `MentionInput` component wrapping a textarea. Parses `@name` on submit, looks up users, creates `Notification` records. Needs to be inserted wherever there's a text input for comments/notes. Use a shared hook `useMentions(workspaceId)` that fetches workspace members once and caches.
**Blocker:** Significant cross-cutting change across item comments, CRM timeline, deals notes.

### D4. Pipeline as own dataset [FUTURE]
**What:** Pipeline view becomes multiple "boards" managing deals, isolated from workspace boards. Each pipeline board has stages (columns) and deals (rows). Recycles board/group/view patterns.
**Architecture plan:** New `Pipeline` model (workspaceId, title) + `PipelineStage` model (pipelineId, title, position, color). Deals get a `pipelineId` FK. Pipeline view renders stages as kanban columns.
**Blocker:** Major schema changes + new UI views.

### D5. Custom deal stages [FUTURE]
**What:** Stages pulled from a custom stage-set defined in the pipeline.
**Dependency:** Requires D4 (Pipeline as own dataset).

### D6. Settings — email function [FUTURE]
**What:** Make email integration actually work (OAuth flow for Gmail/Outlook, sync to CRM timeline).
**Blocker:** Needs Google/Microsoft OAuth app credentials, callback URLs, token storage. Currently stubbed out. Full implementation needed.
